# Action Prioritization — Timing & Ranking Framework

## Purpose

Rank and time-classify action items with explicit rationale, resource requirements, and success metrics. Ensure every recommendation has a clear "why now" or "why then" justification.

## Prioritization Framework: ICE Scoring

Score each action on three dimensions:

| Dimension | Scale | What to Assess |
|-----------|-------|----------------|
| **Impact** (I) | 1-10 | Magnitude of expected outcome on fund/portco value |
| **Confidence** (C) | 1-10 | How certain are we this will work? Evidence base. |
| **Ease** (E) | 1-10 | Inverse of effort, cost, and complexity. 10 = trivial. |

**Priority Score** = (I × C × E) / 10

| Score Range | Priority Level |
|-------------|---------------|
| ≥ 60 | Critical — do immediately |
| 40-59 | High — next 1-3 months |
| 20-39 | Medium — plan for 3-12 months |
| < 20 | Low — backlog or defer |

## Timing Classification

Every action must have a timing category with **explicit rationale** for why it belongs there.

### Immediate (0-30 days)

**Criteria for Immediate** — must meet at least one:
- Active threat to portfolio value (deteriorating covenant, key person departure)
- Time-sensitive opportunity with closing window
- Regulatory deadline or compliance requirement
- Stakeholder commitment already made (LP, board, regulator)

**Rationale template**: "Immediate because [specific trigger]. Delay risk: [what happens if we wait 30+ days]."

### Short-Term (1-3 months)

**Criteria** — important but not time-critical:
- Requires planning, team assembly, or budget allocation before execution
- Dependencies on other actions completing first
- Meaningful impact but no imminent deterioration from delay

**Rationale template**: "Short-term because [why not immediate] and [why not longer]. Dependencies: [what must happen first]."

### Medium-Term (3-12 months)

**Criteria** — strategic and requires sustained effort:
- Multi-phase initiative requiring organizational change
- Market-dependent timing (exit windows, M&A cycles)
- Requires significant investment or hiring

**Rationale template**: "Medium-term because [what makes this a multi-month effort]. Trigger for acceleration: [what would move this to short-term]."

### Long-Term (12+ months)

**Criteria** — structural or market-dependent:
- Major strategic pivots or transformations
- Dependent on external market conditions
- Requires building capabilities that don't exist today

**Rationale template**: "Long-term because [structural reason]. Will accelerate if [specific trigger occurs]."

## Resource Assessment

For each action, document:

| Field | Description |
|-------|-------------|
| Team/skills required | Specific roles or expertise needed |
| Estimated hours | FTE hours or % allocation |
| External cost | Advisory, consulting, legal, technology costs |
| Internal capacity | Can existing team absorb this? Yes/No/Partially |
| Resource conflicts | Does this compete with other high-priority actions for same resources? |

## Success Metrics

Every action must have:

| Field | Requirement |
|-------|-------------|
| Success metric | Quantifiable outcome (revenue, margin, MOIC, risk score change) |
| Baseline | Current state measurement |
| Target | Expected state after execution |
| Checkpoint date | When to measure progress (interim, not just completion) |
| Escalation trigger | What condition indicates the action is failing and needs executive attention |

## "Do Nothing" Analysis

For each recommended action, briefly state the risk of inaction:

"If we don't [action], the expected impact is [quantified downside] within [timeframe]."

This justifies the resource allocation and helps with trade-off decisions.

## Output Format

Present actions in ranked order:

| # | Action | Priority Score | Timing | Rationale (1 sentence) | Owner | Success Metric | Risk of Inaction |
|---|--------|---------------|--------|----------------------|-------|---------------|-----------------|

Follow with:
- Resource conflict analysis (where multiple actions compete for same resources)
- Suggested execution sequence (Gantt-style text timeline)
- Quick wins highlighted (high impact, high confidence, high ease)

## Critical Rules

1. Every timing classification MUST include an explicit rationale
2. Maximum 15-20 actions — focus on vital few, not exhaustive list
3. Resource requirements must be realistic
4. Success metrics must be measurable, not vague ("improve" is not a metric)
5. Flag dependencies between actions explicitly
6. Respect user-specified constraints (budget, capacity, timeline)
7. Highlight quick wins prominently — they build momentum
